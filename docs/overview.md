# Code Interpreter 系列文章全景图

> **知乎专栏：代码会跑起来的那一刻，AI 才算真正懂了计算**

---

## 系列总览

本系列从一个你每天都在用但从未深究的工具出发——**Code Interpreter（代码解释器）**，撕开它光滑的界面，深入那些从未被讲清楚的技术细节：沙箱为什么是安全的？LLM 怎么知道代码跑错了？会话状态是怎么保存的？工业级部署面临哪些工程挑战？

文章分为 **5 个系列，共 22 篇**，从概念到原理，从原理到工程，从工程到手撸实现，层层递进。

---

|---|----------|----------|

| 1-2 | **你的代码是怎么"跑"起来的？——执行管道全解析** | 从用户输入到执行结果的完整数据流 |
| 1-3 | **LLM 与代码执行：两个世界的对话协议** | Function Calling、Tool Use、JSON Schema |
| 1-4 | **状态、会话与上下文：Code Interpreter 的"记忆"机制** | Jupyter Kernel 有状态执行、会话生命周期 |

> **主题定位**：安全工程核心，从威胁建模到硬件隔离

| 2-2 | **Container、gVisor 还是 microVM？——安全边界选型指南** | 五种隔离技术的原理与对比 |
